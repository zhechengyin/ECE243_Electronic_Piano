# 第一阶段进展总结（音频 A + 键盘/VGA B）& 接口对接说明（截至目前）

## 目前我们已经完成的内容（队员 A：音频）

### 1) 音源（Oscillator：fixed-point + wavetable）

* 使用 **32-bit phase accumulator**（`phase`）+ **step**（由频率决定）实现可变频率输出
* 采用 **256 点 sine table**（`sine_table[256]`），运行时仅查表，不做浮点运算
* 核心 API 已实现：

  * `osc_init()`：初始化 wavetable，reset `phase/step`
  * `osc_set_frequency(freq_hz)`：计算 `step = (freq_hz * 2^32) / SAMPLE_RATE`（SAMPLE_RATE=8000）
  * `osc_next_sample()`：`phase += step`，`idx = phase >> 24` 查表返回 `int16_t` sample

### 2) 合成器层（Synth：gate + 频率更新优化）

* 增加 `synth.c` 作为“main 与 oscillator 的中间层”，用于：

  * 维护当前是否发声（gate：`g_note_on`）
  * 维护当前频率（`g_freq_hz`）
  * **只有在 note 从 off→on 或频率变化时**才调用 `osc_set_frequency()`，避免重复计算
* 核心 API 已实现：

  * `synth_init()`
  * `synth_set_note(note_on, freq_hz)`
  * `synth_next_sample()`（note_on=0 返回 0 静音）

### 3) 音频输出驱动（Audio FIFO）

* 已实现 `audio.c`：

  * `audio_write_space()`：读取 Fifospace，取 `min(WSLC, WSRC)`，保证左右声道同步写入

    * WSLC = bits 31..24，WSRC = bits 23..16
  * `audio_write_sample(L, R)`：写 Leftdata/Rightdata
* main loop 采用 batch 写入（例如最多写 32 个 sample/次）以保证连续且为未来 PS/2/VGA 留出时间片

### 4) Key → Note（频率）映射 + glue（piano_engine）

* 已建立 KeyEvent 数据结构（跨模块接口）：

  * `KeyCode`：A/S/D/F/G/H/J/K/L（enum）
  * `KeyEvent`：`{ key, pressed }`
* 已实现 `notes.c` 的 mapping：**KEY_A -> C4(261Hz)**，依次为 D4/E4...（常见 C 调）
* 已实现 `piano_engine.c`：

  * `piano_engine_on_key_event(KeyEvent ev)`：KeyEvent → freq → `synth_set_note(ev.pressed, freq)`

## 关键：我们两部分 combine 的接口（必须遵守）

### ✅ 统一接口：KeyEvent

队员 B 只需要在 PS/2 解析后产出：

* `KeyEvent { key = KEY_?, pressed = 1 }` （按下）
* `KeyEvent { key = KEY_?, pressed = 0 }` （松开）

队员 A 的音频侧只需要消费 KeyEvent，并调用：

* `piano_engine_on_key_event(ev);`

它内部会：

* `freq = note_freq_from_keycode(ev.key);`
* `synth_set_note(ev.pressed, freq);`

### ✅ 音频输出主循环保持不变

无论输入来自 switch 还是 PS/2，最终音频循环始终是：

* `space = audio_write_space();`
* 写入 `space` 个 sample：

  * `s = synth_next_sample();`
  * `audio_write_sample(s, s);`

这样我们集成时只需要“把 KeyEvent 接到 piano_engine”，不会改动音频底层。

# 第二阶段进展更新（PS/2 + VGA 已实现）— 截至 2026/03/11

## 队员 B 已完成的内容

### 1) PS/2 键盘驱动 — `ps2.c` / `ps2.h` ✅

* **硬件读取**：从 PS/2 数据寄存器 `0xFF200100` 轮询读取，通过 RVALID (bit 15) 判断数据有效性
* **扫描码映射**：已实现 Set 2 扫描码 → `KeyCode` 的完整映射：
  * `0x1C→A`, `0x1B→S`, `0x23→D`, `0x2B→F`, `0x34→G`, `0x33→H`, `0x3B→J`, `0x42→K`, `0x4B→L`
* **状态机解析**：`PS2Parser` 结构体 + `ps2_parse_byte()` 实现 2 状态 FSM：
  * 收到 `0xF0`（break code）→ 标记 `break_pending`
  * 收到非 `0xF0` 的有效扫描码 → 输出 `KeyEvent`（`pressed = !break_pending`），然后清除 pending
  * 不支持的键会 reset FSM 但不产生事件，不会卡死
* **辅助函数**：`ps2_key_to_string()` 用于调试打印

### 2) VGA 显示驱动 — `vga.c` / `vga.h` ✅

* **像素绘制**：`vga_put_pixel()` 直接写入 FPGA Pixel Buffer（`0x08000000 + (y<<10) + (x<<1)`），有边界检查
* **图形基础**：`vga_fill_rect()` 填充矩形、`vga_draw_rect()` 画矩形边框、`vga_clear()` 全屏清除
* **字符缓冲**：`vga_char_put()` 写入 80×60 字符缓冲区（`0x09000000`），`vga_char_clear()` 全屏清空
* **琴键布局**：9 个白键，每键 60px 宽 × 200px 高，起始坐标 (50, 140)
* **琴键绘制**：
  * `piano_draw_static()`：绘制完整初始琴键（灰色背景 + 9 个白键 + 字母标签 A–L）
  * `piano_draw_key(key, highlighted)`：单键绘制，高亮 = 红色，正常 = 白色，带黑色边框
  * `piano_handle_key_event(event)`：收到 KeyEvent 后更新内部 `key_state[]`，**只重绘变化的键**（高效局部更新），并刷新标签
* **颜色定义**（RGB565）：黑 `0x0000`、白 `0xFFFF`、红 `0xF800`、灰 `0x8410`

### 3) Testbench 测试程序 ✅

* **`tb_ps2_fpga.c`**：纯 PS/2 硬件测试，读取原始扫描码 + 解析后打印到 JTAG UART，不涉及 VGA/音频
* **`tb_vga_fpga.c`**：PS/2 + VGA 集成测试，启动时画静态琴键，按键事件驱动琴键高亮，不涉及音频
